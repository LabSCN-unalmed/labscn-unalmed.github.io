Modos de texto I

  • dumping y dputing son tiles porque el archivo que resulta es en un formato que se puede corregir, y en caso de que el archivo se degrade, potencialmente se puede recuperar.
  • A diferencia de escribir los resultados en una tabla o en un formato csv, dump y dput mantiene la informacin de metadatos (sacrificando algo de legibilidad), de tal manera que otro usuario no tendra que reconstruir el objeto de nuevo.

Modos de texto II

  • Los modos de texto funcionan mejor para interactuar con programas que llevan el seguimiento de cambios de archivo tipo texto.
  • Este modo sigue la “filosofa linux” en lo que respecta a archivos.
  • Desventaja: No es eficiente en el uso del espacio.

y <- data.frame(a = 1:6, b = letters[1:6])

Interfaces al mundo exterior

  • file, abre una conexin a algn archivo.
  • gzfile, abre una conexin a un archivo comprimido con gzip.
  • bzfile, abre una conexin a un archivo comprimido con bzip2.
  • url, abre una conexin a una pgina web.

Conexin tipo "file"

str(file)
  • description es el nombre del archivo.
  • open es un cdigo que indica
    • r solo lectura.
    • w escritura (inicia un nuevo archivo).
    • a adicin a un archivo.
    • rb, rb, ab, lectura, escritura y adicin en modo binario (nicamente en windows)

Recuperar el valor de la TRM desde enero 1 hasta hoy

require(stringr)
fechas <- str_extract(x, "[0-9]{2}/[0-9]{2}/[0-9]{4}")
valores <- str_extract(x, "[0-9][\\.][0-9]{3},[0-9]{2}")
fechas <- fechas[!is.na(fechas)]
fechas <- as.POSIXct(strptime(fechas, format = "%d/%m/%Y"))
valores <- valores[!is.na(valores)]
valores <- str_replace(valores, "[\\.]","")
valores <- as.numeric(str_replace(valores,",","."))
trm <- data.frame(fechas, valores)
